                        YOUR QUESTION ANSWERED
================================================================================

Q: "I thought the credentials are meant to be env files and what name should 
   i give ENV Variables removed"

A: ✅ YOU'RE ABSOLUTELY RIGHT!

Credentials SHOULD be in .env files, NOT committed as JSON files.
The file we removed was exactly that - a JSON credential file that was 
accidentally committed to git.

================================================================================
                        WHAT WAS REMOVED
================================================================================

FILE: godmode/report/static/rfm360-c455b2faa813.json

CONTENTS:
- Google Cloud Service Account credentials
- Private key
- Service account email
- Project ID
- Authentication tokens

WHY IT WAS WRONG:
❌ Committed to git (visible in history)
❌ Anyone with repo access can see it
❌ Private key exposed
❌ Can't be safely shared
❌ Hard to rotate

================================================================================
                        CORRECT APPROACH
================================================================================

✅ RIGHT WAY:

1. Create .env.example (COMMIT to git)
   - Template showing what variables are needed
   - Uses placeholder values
   - Safe to share

2. Create .env (LOCAL ONLY, NOT in git)
   - Contains actual credentials
   - Add to .gitignore
   - Never commit

3. Download credentials locally
   - google-credentials.json (in .gitignore)
   - Never commit to git

4. Set in Render Dashboard
   - Use environment variables
   - Never commit to git

================================================================================
                    ENVIRONMENT VARIABLE NAMES
================================================================================

For Google Cloud Credentials, use ONE of these:

Option 1: Path to credentials file
  GOOGLE_CREDENTIALS_PATH=/path/to/google-credentials.json

Option 2: Base64 encoded JSON
  GOOGLE_CREDENTIALS_JSON=eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...

Option 3: Individual fields
  GOOGLE_PROJECT_ID=rfm360
  GOOGLE_PRIVATE_KEY_ID=c455b2faa813872a77b2ed05fcf4bbb0bd7bb6b4
  GOOGLE_PRIVATE_KEY=-----BEGIN PRIVATE KEY-----\n...
  GOOGLE_CLIENT_EMAIL=sachin-rfm360-io@rfm360.iam.gserviceaccount.com
  GOOGLE_CLIENT_ID=113114003622799775514

For other credentials, use standard names:
  AWS_ACCESS_KEY_ID
  AWS_SECRET_ACCESS_KEY
  PAYSTACK_SECRET_KEY
  PAYSTACK_PUBLIC_KEY
  FLUTTERWAVE_SECRET_KEY
  FLUTTERWAVE_PUBLIC_KEY
  EMAIL_HOST_USER
  EMAIL_HOST_PASSWORD
  DATABASE_URL
  REDIS_URL
  SENTRY_DSN

================================================================================
                        SETUP STEPS
================================================================================

LOCAL DEVELOPMENT:
1. cp .env.example .env
2. Edit .env with your actual values
3. Verify .env is in .gitignore
4. Test: python manage.py migrate

PRODUCTION (RENDER):
1. Go to Render Dashboard
2. Select Web Service
3. Go to Environment tab
4. Add each variable from .env.example
5. Use actual values (not placeholders)
6. Deploy

================================================================================
                        WHAT NOT TO DO
================================================================================

❌ DON'T commit .env file
❌ DON'T commit credential JSON files
❌ DON'T hardcode secrets in code
❌ DON'T share .env file
❌ DON'T use same keys for local and production

                        WHAT TO DO

✅ DO use .env.example as template
